# 项目分析

将本地或远程项目导入 FlowVision，AI 自动分析代码结构、技术栈、依赖关系，生成可视化架构流程图。

## 支持的导入方式

### 本地目录

通过文件浏览器选择本地项目目录，FlowVision 会：
1. 递归扫描文件结构
2. 识别技术栈和框架
3. 提取模块依赖关系
4. 生成架构可视化图

### GitHub 仓库

输入 GitHub 仓库 URL，自动拉取代码进行分析：

```
https://github.com/username/repository
```

### Gitee 仓库

同样支持 Gitee 仓库：

```
https://gitee.com/username/repository
```

## 分析维度

AI 分析引擎（`@flowvision/analyzer`）会从多个维度提取信息：

| 维度 | 说明 |
|------|------|
| **文件结构** | 目录树、文件类型分布 |
| **技术栈** | 框架、语言、构建工具 |
| **依赖关系** | 模块间的导入/引用链 |
| **设计模式** | 识别常见设计模式 |
| **入口点** | 应用入口和路由结构 |

## 分析结果

分析完成后，生成的流程图包含：

- **模块节点**：每个核心模块对应一个节点
- **依赖连线**：模块间的引用关系
- **分组容器**：按层级归类（前端/后端/共享等）
- **类型标注**：节点类型反映模块角色

可在画布上自由编辑、调整布局、添加注释。
