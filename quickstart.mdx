## 三步启动文档站

1. 安装依赖

```bash
pnpm install
```

2. 启动本地预览

```bash
pnpm dev
```

默认预览地址为 `http://localhost:3000`。保存 MDX 页面或样式文件后会自动刷新。

3. 发布到 GitHub Pages

将变更推送到 `main` 分支后，仓库中的 GitHub Actions 工作流会自动执行构建并发布静态产物。

## 发布前检查

<details className="fv-details">
  <summary>确认本地构建可通过</summary>

在提交前先运行 `pnpm build`，确认所有 MDX 页面、静态资源和站点配置都能正常生成。

:::tip
如果仓库名称发生变化，可以通过环境变量 `DOCS_BASE_URL` 覆盖默认的发布子路径。
:::

</details>

<details className="fv-details">
  <summary>启用 GitHub Pages</summary>

在仓库设置中将 Pages 的 Source 设置为 GitHub Actions。首次推送后，工作流会自动生成部署环境并发布站点。

</details>

<details className="fv-details">
  <summary>调整站点标题与主题色</summary>

站点标题、导航和底部链接位于 `docusaurus.config.ts`，主题样式位于 `src/css/custom.css`。

</details>
