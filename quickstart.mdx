## 三步启动文档站

<Steps>
  <Step title="安装依赖">

```bash
pnpm install
```

  </Step>
  <Step title="启动本地预览">

```bash
pnpm dev
```

默认预览地址为 `http://localhost:3000`。保存 MDX 页面或样式文件后会自动刷新。

  </Step>
  <Step title="发布到 GitHub Pages">

将变更推送到 `main` 分支后，仓库中的 GitHub Actions 工作流会自动执行构建并发布静态产物。

  </Step>
</Steps>

## 发布前检查

<AccordionGroup>
  <Accordion icon="copy" title="确认本地构建可通过">
    在提交前先运行 `pnpm build`，确认所有 MDX 页面、兼容组件和静态资源都能正常生成。

    <Tip>如果仓库名称发生变化，可以通过环境变量 `DOCS_BASE_URL` 覆盖默认的发布子路径。</Tip>
  </Accordion>
  <Accordion icon="github" title="启用 GitHub Pages">
    在仓库设置中将 Pages 的 Source 设置为 GitHub Actions。首次推送后，工作流会自动生成部署环境并发布站点。
  </Accordion>
  <Accordion icon="palette" title="调整站点标题与主题色">
    站点标题、导航和底部链接位于 `docusaurus.config.mjs`，卡片样式与品牌配色位于 `src/css/custom.css`。
  </Accordion>
</AccordionGroup>
