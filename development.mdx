- Node.js 20 或更高版本
- pnpm 9 或更高版本
:::

## 本地开发

1. 安装依赖

```bash
pnpm install
```

2. 启动开发服务器

```bash
pnpm dev
```

本地预览默认运行在 `http://localhost:3000`。

3. 验证静态构建

```bash
pnpm build
pnpm serve
```

先生成 `build/`，再用本地静态服务模拟 GitHub Pages 的实际产物。

## 静态资源与路径

本项目通过 `staticDirectories` 直接复用现有的 `images/` 与 `logo/` 目录，不再依赖额外的资源复制脚本。

## 常见问题

<details className="fv-details">
  <summary>构建后图片或链接在 GitHub Pages 上 404</summary>

这通常是 `baseUrl` 与仓库名不一致导致的。项目会默认根据 GitHub 仓库名生成项目页子路径；如果你部署到自定义路径，请设置 `DOCS_BASE_URL`。

</details>

<details className="fv-details">
  <summary>MDX 页面渲染异常</summary>

当前项目已切换为原生 Docusaurus 语法。如果你新增了 JSX、Tabs 或 details 结构，先运行 `pnpm build` 检查 MDX 编译结果。

</details>

<details className="fv-details">
  <summary>GitHub Actions 构建失败</summary>

先在本地运行 `pnpm install` 和 `pnpm build`。如果本地通过，再检查仓库是否已启用 GitHub Pages 的 GitHub Actions Source。

</details>
